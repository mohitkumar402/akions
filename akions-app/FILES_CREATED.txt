🚀 QUICK START:

1. Navigate to project:
   cd ./mohit/akions/akions-app

2. Setup environment (first time only):
   cp .env.example .env
   # Edit .env with your settings

3. Start all services:
   docker compose up -d --pull always

4. Access services:
   ✓ Backend API:  http://localhost:3000
   ✓ Frontend Web: http://localhost:19006
   ✓ MongoDB:      localhost:27017

5. View logs:
   docker compose logs -f

6. Stop services:
   docker compose down

📋 USEFUL COMMANDS:

Service Management:
  docker compose ps                 - Show running containers
  docker compose logs -f            - Follow logs live
  docker compose exec backend sh    - Shell into backend
  docker compose restart            - Restart all services
  docker compose down -v            - Stop and remove volumes

Database:
  docker compose exec mongodb mongosh admin -u admin -p password

Seeding:
  docker compose exec backend npm run seed:blogs
  docker compose exec backend npm run seed:internships
  docker compose exec backend npm run seed:products
  docker compose exec backend npm run create:admin

Debugging:
  docker compose ps                 - Check service health
  docker compose logs [service]     - View specific logs
  docker system df                  - Disk usage
  docker stats                      - Resource usage

🔒 SECURITY NOTES:

⚠️  Important for Production:

1. Change JWT_SECRET to a strong random value
2. Change MongoDB credentials (MONGODB_PASSWORD)
3. Don't commit .env file to Git
4. Use HTTPS in production
5. Review CORS_ALLOWED_ORIGINS configuration
6. Consider using managed services (MongoDB Atlas)
7. Implement rate limiting on API endpoints
8. Regular security audits and dependency updates

❓ TROUBLESHOOTING:

Q: Services won't start?
A: Check logs: docker compose logs

Q: Port already in use?
A: Edit .env and change port numbers

Q: MongoDB won't connect?
A: Wait 30s for startup, check logs: docker compose logs mongodb

Q: Want to reset everything?
A: docker compose down -v (WARNING: deletes all data)

For more help, see DOCKER_GUIDE.md
